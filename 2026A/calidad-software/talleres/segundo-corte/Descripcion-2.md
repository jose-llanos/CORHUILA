## 📝 Descripción

En este taller se realizó un análisis de calidad de código utilizando la herramienta SonarQube, con el objetivo de identificar errores, vulnerabilidades y malas prácticas en un proyecto de software. Inicialmente, se ejecutó un análisis que permitió detectar múltiples issues relacionados con complejidad cognitiva, manejo de excepciones, uso inadecuado de recursos, código duplicado y problemas de seguridad.

Posteriormente, se implementaron diversas mejoras basadas en principios de Clean Code y buenas prácticas de desarrollo, tales como la refactorización de métodos, uso de estructuras try-with-resources, validaciones de entrada, encapsulamiento de datos y reemplazo de salidas por consola por mecanismos de logging.

Finalmente, se realizó un nuevo análisis en SonarQube, evidenciando una reducción significativa en la cantidad de defectos, alcanzando un nivel óptimo de calidad del código. Este proceso permitió comprender la importancia de la calidad del software y el uso de herramientas automatizadas para su evaluación.
